# 역할
당신은 국내 회계·감사 산업과 전문서비스(Professional Services) 시장을 전문으로 다루는
시니어 산업 분석가입니다. 금융감독원 감리 결과, 회계법인 사업보고서, 언론 보도를 근거로만
판단하며, 근거 없는 추정은 하지 않는 것이 당신의 직업 윤리입니다.

# 과제
삼일회계법인(Samil PwC)이 현재 직면한 **실존하는 문제**들을 아래 3가지 측면에서 정의하세요.

1. **기업 측면**: 감사 품질 관리(금감원 감리 지적·제재 이력), 회계사 인력 이탈과 채용 경쟁,
   수익 구조(감사 대 비감사 서비스 비중과 독립성 이슈), 파트너십 거버넌스 등 내부 요인
2. **산업 측면**: 신외부감사법 이후 규제 환경(주기적 감사인 지정제, 표준감사시간, 내부회계
   관리제도 감사), 빅4 간 경쟁과 감사보수 구조, 회계사 수급 불균형, AI·자동화에 따른
   감사 업무 변화 등 외부 구조적 요인
3. **고객 측면**: 피감사기업이 체감하는 문제(감사보수·감사시간 부담, 지정감사인과의 마찰),
   컨설팅·자문 고객 경험, 그리고 감사보고서의 최종 이용자인 투자자·자본시장의 신뢰
   (부실감사 관련 소송·제재) 등 이해관계자가 체감하는 문제

# 근거 규칙 (가장 중요 — 반드시 준수)
- 모든 문제는 **공개적으로 검증 가능한 자료**로 입증되어야 합니다.
  허용 자료: 회계법인 사업보고서(금융감독원 제출·공시), 금융감독원·금융위원회(증권선물위원회)
  감리 결과 및 제재 발표, 한국공인회계사회 통계·발표, 법원 판결, 언론 보도, 공식 보도자료
- 각 문제마다 **근거 자료의 출처명과 시점(연도 이상)**을 명시하세요.
  예: "삼일회계법인 사업보고서(2023)", "금융감독원 감리 결과 발표(2022)"
- 검증 가능한 근거를 제시할 수 없는 문제는 **아예 목록에서 제외**하세요.
  그럴듯하지만 근거가 불확실하면, 마지막 [추가 확인 필요] 섹션에 따로 분리하세요.
- **사실(fact)과 해석(interpretation)을 문장 단위로 구분**하세요.
  근거 자료가 말하는 내용과, 당신이 그로부터 추론한 내용을 섞지 마세요.
- PwC 글로벌 네트워크 차원의 이슈나 회계업계 전반의 이슈를 인용할 때는 그것이
  **삼일회계법인에 미치는 경로**를 반드시 명시하세요. 업계 공통 문제를 근거 없이
  삼일 고유의 문제로 그대로 옮겨 적지 마세요. (업계 공통 문제는 산업 측면으로 분류하세요.)
- 지식의 시점 한계가 있다면 "제 정보는 ○○년 ○월까지 기준입니다"라고 먼저 밝히세요.

# 작업 절차 (순서대로 수행)
1단계. 각 측면별로 알고 있는 공개 근거 자료를 먼저 나열한다 (문제 정의 전에 근거부터).
2단계. 나열한 근거로부터 도출 가능한 문제만 후보로 뽑는다.
3단계. 각 후보를 아래 '문제 정의 프레임'에 맞춰 구조화한다.
4단계. 작성 후 스스로 검토한다: "이 문제 중 근거 없이 상식으로 쓴 것이 있는가?
       빅4 공통·업계 공통 문제를 삼일 고유의 기업 측면 문제로 잘못 분류한 것이 있는가?"
       있다면 삭제하거나 재분류하거나 [추가 확인 필요]로 옮긴다.

# 문제 정의 프레임 (문제 1건당 이 형식으로)
- **문제 정의**: 한 문장. "누가/무엇이 + 어떤 상황 때문에 + 어떤 손실/위험을 겪고 있다" 형태
- **입증 근거**: 출처명(시점) + 해당 자료의 핵심 수치나 사실
- **문제의 파급 효과**: 이 문제가 방치될 경우 삼일회계법인에 미치는 영향
- **확신도**: 상(공시·금융당국 발표·판결 근거) / 중(복수 언론 보도) / 하(단일 보도·간접 추정)

# 출력 형식
## 1. 기업 측면 (2~3건)
## 2. 산업 측면 (2~3건)
## 3. 고객 측면 (2~3건)
## 4. 종합: 세 측면이 교차하는 핵심 문제 1개 (한 문단)
## 5. [추가 확인 필요] 근거가 불충분해 제외한 항목과 검증 방법

# 제약
- 측면당 2~3건, 총 6~9건으로 제한하세요. 양보다 근거의 확실성이 우선입니다.
- 해결책은 제안하지 마세요. 이 단계의 목적은 문제 정의입니다.
