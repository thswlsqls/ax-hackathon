참여 기업의 공개·검증 가능한 문제를 해결하는 Codex 플러그인
이 과제는 여러분이 선택한 기업을 대상으로 합니다. 그 기업, 그 기업의 산업, 또는 그 기업의 고객이 겪는 실제 문제를 공개 자료로 입증하고, 이를 해결하는 Codex 플러그인을 제출하세요.

기업을 여러 개 선택했다면, 각 기업마다 별도의 플러그인을 만들어 기업별로 따로 제출해야 합니다. 하나의 제출물로 여러 기업을 함께 다룰 수는 없습니다.

Codex 플러그인 작성·실행 방법은 OpenAI 공식 문서를 참고하세요.

📚 Codex 공식 문서

플러그인 만들기
https://developers.openai.com/codex/plugins/build

무엇을 제출하나요
1. submission.zip
submission.zip
├── src/
│   ├── .codex-plugin/plugin.json
│   ├── skills/<이름>/SKILL.md
│   ├── .mcp.json
│   └── 그 밖의 실행 코드와 설정 파일
├── README.md
└── logs/
전체 플러그인 루트가 src 안에 있어야 합니다.
src/.codex-plugin/plugin.json은 필수입니다.
plugin.json 외에, 플러그인이 실제로 동작하도록 만드는 구성 요소를 하나 이상 넣어야 합니다. 스킬(SKILL.md)이 가장 일반적이며, 필요하면 MCP 서버(.mcp.json)나 직접 작성한 실행 코드를 추가할 수 있습니다.
src 안의 파일·폴더 구조는 자유롭게 가져가되, Codex 플러그인으로 실행이 되어야 합니다.
로그 (logs 폴더)
플러그인을 만들면서 AI와 주고받은 대화 로그를 logs 폴더에 넣어 함께 제출합니다.

편집·발췌 없이 전체를 그대로 제출합니다.
제출 가능한 파일 형식: md, txt, json, jsonl
로그·플러그인·질문지 사이의 정합성을 함께 봅니다. 서로 어긋나면 평가에 불리하게 작용합니다.
📦 로그 수집 도구
log-hooks.zip 다운로드
https://github.com/jocoding-ax-partners/axwar/releases/download/v1.0/log-hooks.zip

로그를 모으는 방법은 사용하는 도구에 따라 다릅니다.

Claude Code · Codex CLI - 위 파일을 내려받아 작업 폴더 루트에 압축을 해제하면, 포함된 훅이 로그를 logs 폴더에 자동으로 저장합니다. Codex CLI는 최초 실행 시 훅 신뢰 여부를 묻는데, 로그 저장을 위해 승인하세요.
그 밖의 도구(Cursor · ChatGPT 웹 등) - 전체 대화를 편집·발췌 없이 텍스트 형식(txt, md, json 등)으로 저장하여 logs 폴더에 넣으세요. 이미지·PDF처럼 텍스트 추출이 불가능한 형식은 미제출로 간주합니다.
주의.
AI 채팅에 API 키, 비밀번호, 토큰 등 비밀정보를 입력하지 마세요. 로그에 그대로 남습니다.

2. 질문 5문항
2.1. 무엇을, 누가, 어떤 상황에서 쓰나요?
2.2. 왜 이 문제를 선택했나요?
2.3. 플러그인은 어떻게 작동하나요?
2.4. AI를 어떻게 활용했나요?
2.5. 어떻게 검증했나요?

채점 안내
예선 과제는 AI와 심사자가 함께 평가합니다.

주의사항
⚠️ 실격 주의
로그 파일을 원본 그대로가 아닌 사후 가공(편집·발췌·삭제)하여 제출할 경우 실격 처리됩니다.

출처는 공개 자료로 AI가 확인할 수 있어야 합니다.
기업 내부의 비공개 정보, 확인할 수 없는 사적인 경험, 출처 없는 숫자는 근거로 인정되지 않습니다.
❗️ 제출하신 예선 제출물은 본선 심사에 활용될 수 있습니다.
