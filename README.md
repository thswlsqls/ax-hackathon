# AX 해커톤 예선 제출물 모음

참여 기업의 **공개·검증 가능한 문제**를 해결하는 Codex 플러그인 제출물 저장소입니다.
예선 과제 본문은 [`docs/예선문제.txt`](docs/예선문제.txt)를 참고하세요.

> 이 과제는 선택한 기업을 대상으로, 그 기업·산업·고객이 겪는 실제 문제를 공개 자료로
> 입증하고 이를 해결하는 Codex 플러그인을 제출하는 것입니다. 기업을 여러 개 선택하면
> **기업마다 별도의 플러그인**으로 따로 제출해야 하므로, 이 저장소는 기업별 제출물을
> 각각 독립된 디렉터리로 관리합니다.

## 제출물 목록

| 기업 | 플러그인 | 해결하는 공개 문제 | 위치 |
|------|----------|---------------------|------|
| 카카오페이증권 | `mts-stability-guard` | 반복되는 MTS·해외주식 전산장애 재발방지 (공개 장애 이력에서 실패 패턴 추출 → 릴리즈 전 회귀·부하·장애주입 시나리오와 안정성 체크리스트 생성) | [카카오페이증권/submission](카카오페이증권/submission/README.md) |
| 무신사 | `musinsa-contract-risk-scanner` | 입점·납품 계약서의 대규모유통업법·공정거래법상 리스크 조항 탐지 → 위험등급·근거 법조문·대체 문구 제안 (사전 검토 우선순위 큐) | [무신사/submission](무신사/submission/README.md) |
| 삼일회계법인 | `samil-independence-screening` | 감사대상회사와 비감사용역 간 독립성 훼손 위협 1차 triage (사용자 제공 룰셋 기반 금지용역 판정) | [삼일회계법인/submission](삼일회계법인/submission/README.md) |

각 제출물은 독립적으로 실행·평가되며, 상세한 실행 방법·문제 정의 근거·예선 5문항 답변은
해당 디렉터리의 `README.md`와 `docs/`에 있습니다.

## 저장소 구조

```
ax-hackathon/
├── docs/예선문제.txt        # 예선 과제 본문 (제출 요건·채점 안내)
├── 카카오페이증권/submission/  # 제출물 1 — MTS 전산장애 재발방지
├── 무신사/submission/          # 제출물 2 — 계약서 리스크 스캐너
├── 삼일회계법인/submission/    # 제출물 3 — 독립성 스크리닝
├── tools/                    # 로그를 submission/logs로 저장하는 보조 스크립트
└── README.md
```

각 `submission/` 디렉터리는 예선 제출 규칙을 따릅니다.

- 전체 플러그인 루트가 `src/` 안에 위치합니다.
- `src/.codex-plugin/plugin.json`(필수 매니페스트)과, 플러그인이 실제로 동작하도록
  만드는 구성 요소(스킬 `SKILL.md`, 필요 시 `.mcp.json`·실행 코드)를 포함합니다.
- AI와 주고받은 대화 로그는 `logs/`에 **편집·발췌 없이 원본 그대로** 보관합니다.

## 제출 원칙

- **공개·검증 가능성**: 문제 정의 근거는 AI가 확인 가능한 공개 자료(보도·공시 등)로만
  제시하며, 각 제출물 `docs/`에 출처 URL을 명시합니다.
- **로그 정합성**: 로그·플러그인·질문지 사이의 정합성을 유지하며, 로그를 사후 가공하지
  않습니다.
- **비밀정보 배제**: API 키·비밀번호·토큰 등은 대화·산출물에 남기지 않으며,
  실행 입력/출력은 sanitized 데모 스냅샷만 저장소에 포함합니다.
