# 실패 패턴 레퍼런스

공개 장애 이력을 분류하는 4개 패턴과 재발방지 설계 매핑. 패턴 분류는 사실(공개 원인 인용)에 기반하고, 권고 설계는 해석이다.

| 패턴 키 | 정의 | 대표 공개 사례 | 우선 방어 설계 |
|---|---|---|---|
| `us_market_open_peak` | 미국 정규장 개장 직후 트래픽 폭주로 접속 불능 | 2025-10-07 개장 직후 앱 접속 불능 (다음 2025.10) | 스파이크 부하 테스트, 오토스케일·큐잉 폴백, 개장 전 예열 |
| `external_broker_dependency` | 현지 중개사/외부 API 장애로 체결 불가 | 2025-10-08 드라이브웰스 장애로 미국주식 체결 불가 (MBC/서울신문 2025.10) | 장애주입(chaos), 서킷브레이커·재시도, 대체 체결경로·자동 공지 |
| `internal_system` | 내부 시스템 장애로 주문 지연 | 2025-10-10 00:33 약 40분간 주문 체결 불가, 같은 날 09:00 약 30분간 시스템 응답 지연(서로 다른 두 사건, 다음 2025.10) | 주문경로 회귀 테스트, 배포 전 스모크·자동 롤백 |
| `overseas_latency` | 해외주식 서비스 지연 | 2023-07-03 약 40분 지연 (연합뉴스, 기사 송고 2023-07-04) | 지연 주입 테스트, 알림·보상 트리거, 분산 트레이싱 |

## 이력 스키마
`date`는 기사 송고일이 아닌 한국시간 기준 장애 발생일을 `YYYY-MM-DD` 형식으로 기록한다. 같은 날 여러 사건은 `id`와 요약에 발생 시간을 포함해 구분한다.

```json
{
  "meta": { "subject": "...", "sources": ["..."] },
  "annual_counts": { "basis": "...", "counts": {"2022": 4}, "source": "..." },
  "incidents": [
    {"id": "...", "date": "YYYY-MM-DD", "service": "...",
     "pattern": "us_market_open_peak|external_broker_dependency|internal_system|overseas_latency",
     "duration_min": 40, "cause": "...", "source": "..."}
  ]
}
```
